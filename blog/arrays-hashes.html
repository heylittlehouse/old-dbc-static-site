Arrays and Hashes

The difference

14 September 2015

An array is an ordered, indexed collection of objects. An array can contain anything--strings, numbers, other arrays, hashes, etc. Each element is indexed from zero, up.

A hash is a collection of key-value pairs. What this means is that each element will contain two sub elements that are paired because of some inherent relationship between them. For example, an element with the key ":address" might have the value of "155 W. 60th Street"

You might use an array if you want to access your elements by index number. If that works fine, or you specifically need to cycle through the number of elements in your grouping, then use an array. If you have specific requirements for what you need to look up your element, like if only a specific word will do for accessing your information, then using a hash is the way to go.
